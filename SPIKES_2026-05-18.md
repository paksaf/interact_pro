# Six spikes — 2026-05-18 (showcase-gap closure)

All six items the audit flagged as missing now have working scaffolds in code. Each spike lands as a self-contained module with a stable public surface — call-site integration in the existing screens is a separate, smaller PR and is called out below per spike.

## A — Originator-notify on sign  ✅ shipped

| | |
|---|---|
| New | `lib/features/signatures/data/signature_notifier.dart` |
| Schema | `PdfDocuments.originatorEmail`, `PdfDocuments.originatorPhone` (Drift v7→v8) |
| Wired | `SignatureRepository.signDocument()` calls `_safeNotify(...)` post-insert |
| TODO | `IncomingFileListener` must capture the sender's contact and write it to the `PdfDocuments` row at import. Today it captures the file only. ~30 LOC in `incoming_file_listener.dart`. |

Notification routes through **Comms Hub** (`connect.interactpak.com/api/comms/send`) not direct Resend — that's the same hub Sahulat/FleetOps/Rewards already use, so SMS fallback (Baileys → capcom6 → Twilio) is handled for free.

## B — Admin document browser  ✅ shipped (server-side)

| | |
|---|---|
| Server route | `GET /api/admin/users/:id/documents` (list) |
| Server route | `GET /api/admin/users/:id/documents/:docId/download` (stream) |
| User opt-in | `POST /api/auth/me/admin-doc-consent { allow: true }` — defaults OFF |
| User audit | `GET /api/auth/me/admin-access-log` — every admin read visible to the user |
| Schema | `005_admin_doc_access.sql` — `users.admin_doc_access_allowed`, `users.admin_doc_consent_at`, `sync_documents` index, `audit_log` |
| TODO | Flutter UI: add a "Documents" tab to `AdminScreen` calling the two routes above; add a privacy toggle to the user's Settings → Privacy screen that calls `admin-doc-consent`. ~150 LOC. |

Privacy contract baked in:
  - **default OFF** (admins see nothing until user opts in)
  - **14-day consent TTL** — silently expires; admin gets 403 until user re-opts-in
  - **every read is audit-logged** with filename + size; user pulls their own log

### 2. Smoke each new endpoint

```bash
# Health-aggregator already covers /api/zeka/ai etc. New surface:
JWT=...  # any user's bearer

# E — convert. Use a small docx as the test payload.
curl -sS -o /tmp/out.pdf -w 'HTTP %{http_code}  X-Cache: %header{x-cache}\n' \
  -H "Authorization: Bearer $JWT" \
  -F "file=@test.docx;type=application/vnd.openxmlformats-officedocument.wordprocessingml.document" \
  https://pro.interactpak.com/api/convert/to-pdf
file /tmp/out.pdf  # → "PDF document, version 1.x"

# B — admin doc list (uses a separate admin JWT).

  https://pro.interactpak.com/api/admin/users/$USER_ID/documents | jq .
# Expect 403 with "User has not opted in" until that user POSTs
# /api/auth/me/admin-doc-consent.

# User consent toggle:
curl -sS -X POST -H "Authorization: Bearer $JWT" \
  -H 'Content-Type: application/json' \
  -d '{"allow": true}' \
  https://pro.interactpak.com/api/auth/me/admin-doc-consent | jq .
```
