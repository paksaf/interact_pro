### 2. OTP email operational diagnostic (item 3 root cause)

```bash
ssh interact 'set -e
  echo "=== Resend key present? ==="
  sudo grep "^RESEND_API_KEY=" /etc/interact/pro-api.env | sed "s/=.*/=<set>/" || echo "MISSING"

  echo "=== send.interactpak.com verified? ==="
  KEY=$(sudo grep "^RESEND_API_KEY=" /etc/interact/pro-api.env | cut -d= -f2-)
  curl -sS -H "Authorization: Bearer $KEY" https://api.resend.com/domains \
    | python3 -c "import sys,json;d=json.load(sys.stdin); [print(x[\"name\"],x[\"status\"]) for x in d.get(\"data\",[])]" 2>/dev/null \
    | grep -E "send\.interactpak\.com|interactpak\.com" || echo "NOT FOUND in workspace"

  echo "=== Recent OTP send logs ==="
  journalctl -u interact-pro-api --since "1 hour ago" --no-pager 2>/dev/null \
    | grep -iE "Resend|OTP delivery|sendOtp" | tail -20

  echo "=== DMARC record on interactpak.com ==="
  dig +short TXT _dmarc.interactpak.com | head -3
'
```

Action based on output:
  - `MISSING` Resend key → add it to `/etc/interact/pro-api.env`, restart service.
  - Domain not in workspace → log into [resend.com/domains](https://resend.com/domains) (paksaf workspace), add `send.interactpak.com`, paste the DNS records into Hetzner DNS, wait for verification.
  - Recent logs show `Resend 422 / 403` → the from-address domain isn't verified (see above).
  - No DMARC TXT → add `_dmarc.interactpak.com TXT "v=DMARC1; p=none; rua=mailto:dmarc@interactpak.com"`.

## Smoke tests

### Item 3 — OTP delivery failure surfaces

```bash
# Trigger a deliberate failure by sending an obviously bad email
curl -sS -X POST https://pro.interactpak.com/api/auth/otp/request \
  -H 'Content-Type: application/json' \
  -d '{"email":"no-such-mailbox-12345@invalid-domain-test.example"}' \
  -w '\nHTTP %{http_code}\n'
```

Expect: `HTTP 502` with `delivered: false` and an `error` field. (Pre-fix: HTTP 200 with no failure signal.)

### Items 1 + 5 — TopBar layout + reading-timer

Open a long-titled PDF and a short-titled PDF. Back-arrow should sit left, title should ellipsize, and the chip + bookmark + TTS + spread-toggle group should sit at the right edge of both. After ~20s of reading, force-quit the app (swipe from recents), reopen — the reading-time chip should reflect the elapsed time (was previously lost on force-kill).

### Item 2 — Karaoke

Settings → Read aloud → enable "Highlight spoken words." Open any PDF with extracted text, tap the speaker icon, watch the KaraokeStrip. Words should now advance in step with playback on both System TTS and Piper/Kokoro/eSpeak.

### Item 6a — Signature presets

Sign sheet → save 6 presets in quick succession. The 1st (oldest) should silently evict; you should still see 5 presets afterward, newest at the top. Close + reopen the app — the 5 presets should persist.

### Item 6b — IAP server verify

Sandbox test on iOS or Play Console internal-test on Android. Trigger a purchase. In the pro-api logs you should see a POST to `/api/iap/verify` with `verifier='apple'` (or `'google'`) and `verified=true`. Confirm in DB:

```bash
ssh interact 'psql $DATABASE_URL -c "SELECT platform, product_id, verified, verifier FROM iap_purchases ORDER BY created_at DESC LIMIT 5"'
```

If Apple/Google creds aren't configured yet (`APPLE_SHARED_SECRET` or `GOOGLE_PLAY_SERVICE_ACCOUNT` missing), `/api/iap/verify` returns 402 with `verifier-not-configured`. For staging-only override, set `DEV_ALLOW_UNVERIFIED_IAP=1` in `/etc/interact/pro-api.env`. **Never** set that on prod.
